Before You Start

2 free accounts needed

1
Create a GitHub account → github.com

GitHub stores your code online. Think of it as Google Drive for code. It's free.

2
Create a Vercel account → vercel.com

Click "Sign Up with GitHub" — use your GitHub account. Don't create a separate password.

3
Install Git on your computer

Download from git-scm.com/downloads. After installing, open Terminal (Mac/Linux) or Command Prompt (Windows) and run:

git --version
# Should show something like: git version 2.45.0
Use the same email address for both GitHub and Vercel — it keeps things simple.
1

Upload Code to GitHub

~10 minutes

1A – Create a new repository on GitHub

1
Go to github.com → Click the green "New" button

Find it on the left sidebar. Or click the "+" icon top-right → "New repository".

2
Fill in the repository name 
Repository name: chhatrapati-udyog-samuh
Visibility:       Public

Do NOT tick "Add README" or "Add .gitignore" — your project already has them.

3
Click "Create repository"

GitHub shows a page with setup instructions. Keep this page open — you need the URL shown there.

4
Copy the repository URL

It looks like: https://github.com/YOUR-USERNAME/chhatrapati-udyog-samuh.git

1B – Push your project files from Terminal

Open Terminal (Mac/Linux) or Command Prompt (Windows). Navigate to your project folder:

# Windows example (adjust to where your folder is):
cd C:\Users\YourName\Downloads\chhatrapati-static

# Mac/Linux example:
cd ~/Downloads/chhatrapati-static

Then run these commands one by one:

# Step 1: Initialize git in this folder
git init

# Step 2: Add all files
git add .

# Step 3: Save them with a message
git commit -m "first commit"

# Step 4: Connect to your GitHub repository (paste YOUR URL)
git remote add origin https://github.com/YOUR-USERNAME/chhatrapati-udyog-samuh.git

# Step 5: Set branch name to main
git branch -M main

# Step 6: Upload to GitHub
git push -u origin main
GitHub may ask for your username and password. If password fails, you need a Personal Access Token. Go to: GitHub → Settings → Developer Settings → Personal access tokens → Tokens (classic) → Generate new token → tick "repo" → Copy the token → use it as your password.
After pushing, refresh your GitHub page. You should see all your files listed (index.html, css/, js/, images/). If yes — this step is done!

Common GitHub errors

❌ "remote origin already exists"
Fix: git remote remove origin → then redo the git remote add origin ... line.
❌ "src refspec main does not match any"
Fix: You skipped the commit step. Run git add . then git commit -m "first commit" first.
❌ "Authentication failed"
Fix: Use a Personal Access Token as your password (see warning above). Normal passwords no longer work in Git terminal.

Deploy to Vercel

~10 minutes

Vercel reads your GitHub repository and builds your website automatically. Every time you push new code to GitHub, Vercel redeploys automatically.
1
Go to vercel.com → Click "Add New Project"

You should be already logged in via GitHub.

2
Import your GitHub repository

Find chhatrapati-udyog-samuh in the list → Click "Import".

If you don't see it, click "Adjust GitHub App Permissions" and grant access to your repository.

3
Configure the project settings

Vercel should auto-detect everything correctly. Verify these settings:

Framework Preset:  Other  (NOT Next.js or React)
Root Directory:    ./     (leave as default)
Build Command:     leave empty
Output Directory:  leave empty  (or ./)
This is a static HTML website — there is NO build step. If Vercel auto-detects a framework, change Framework Preset to "Other".
4
Click "Deploy"

Vercel will upload and deploy your files. This takes about 30–60 seconds.

5
You'll see a congratulations screen with your live URL

Your website is live! The URL looks like:

https://chhatrapati-udyog-samuh.vercel.app
Click "Visit" to open your live website. Share this URL with customers!

Common Vercel errors

❌ Website shows blank white page or "404 Not Found"
Fix: Vercel Dashboard → Your project → Settings → General → Framework Preset → Change to Other → Redeploy.
❌ "No Output Directory named 'public'" error
Fix: Vercel Dashboard → Settings → Build & Output Settings → Output Directory → delete the value (leave blank) → Redeploy.
❌ CSS/JS not loading (broken design)
Fix: Make sure your folder structure is correct — css/style.css, js/data.js, js/app.js — and all files were pushed to GitHub. Run git add . && git commit -m "fix" && git push.
❌ Can't find repository during import
Fix: Click "Configure GitHub App" or "Adjust permissions" in Vercel → Grant access to "All repositories" → Retry import.

Update the Website Later

For content changes (plots, photos, contacts)

Only edit js/data.js — no other file needs to change.

1
Open js/data.js in any text editor (Notepad, VS Code, etc.)

Find the value you want to change — plot count, phone number, etc.

2
Make the change and save the file 
# Example: reduce available plots from 15 to 12
availablePlots: 12,   // changed from 15
3
Push to GitHub (Vercel auto-deploys) 
git add .
git commit -m "updated available plots for shiv nagar"
git push
4
Wait 1–2 minutes → Refresh your Vercel URL

Changes are live! Vercel redeploys automatically on every push.

For adding photos

1
Copy photo to the correct folder

शिव नगर → images/projects/shiv-nagar/photo1.jpg
जिजाऊ नगर → images/projects/jijau-nagar/photo1.jpg

Use simple filenames — no Marathi, no spaces. Good: photo1.jpg, front.jpg. Bad: शिव नगर फोटो.jpg
2
Add the filename in data.js photos array 
photos: [
  'images/projects/shiv-nagar/photo1.jpg',
  'images/projects/shiv-nagar/photo2.jpg',
],
3
Push to GitHub 
git add .
git commit -m "added photos for shiv nagar"
git push
5

Redeploy Summary

Every time you want to update the live website, these 3 commands are all you need:

git add .
git commit -m "what you changed"
git push

# Vercel detects the push and auto-deploys within 1–2 minutes.
# No manual steps needed on Vercel dashboard.
You can also trigger a manual redeploy from Vercel Dashboard → Deployments → "Redeploy" — useful if something looks wrong.

Custom Domain (Future)

When you want yourname.com instead of vercel.app

A custom domain like chhatrapatiudyog.in costs about ₹700–₹900/year. The hosting on Vercel remains free forever.
1
Buy a domain from GoDaddy, BigRock, or Hostinger

Search for a .in domain — cheap and looks professional for Indian businesses.

2
In Vercel: Project → Settings → Domains → Add Domain

Type your domain name (e.g. chhatrapatiudyog.in) and click Add.

3
Vercel shows DNS records — add them in your domain registrar

Log in to GoDaddy/BigRock → DNS Management → Add the records shown by Vercel (usually a CNAME or A record).

4
Wait 10–60 minutes for DNS to propagate

Your domain will point to your Vercel website. Vercel also adds free HTTPS (https://) automatically.
